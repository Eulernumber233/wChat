# wChat 客户端 UI 功能需求文档

> 本文档面向客户端 UI 重绘工作，列出**所有已有完整后端实现、必须在新 UI 中保留**的功能入口。
>
> 范围说明：
> - 只列**功能**（用户能完成什么），不规定布局、配色、控件类型、交互细节。
> - 已实现 = 槽函数有实质逻辑、能与服务端正常通信、能改变 UI/数据状态。
> - 仅有占位按钮、空槽、TODO 注释、未连线信号的功能**不在本表内**。
> - 新 UI 可以重新组合页面边界、改变跳转方式、合并或拆分入口，但每条功能都必须有对应的可用 UI 入口。
>
> 配套参考：[CLAUDE.md](../CLAUDE.md) §6（ReqId 总表）、§8（调用链）。

---

## 1. 顶层页面构成

客户端目前由 4 个一级页面组成。新 UI 可以改变它们的呈现形式（窗口 / Tab / 路由皆可），但这 4 个状态必须存在：

| 页面 | 用途 |
|---|---|
| 登录页 | 用户输入邮箱+密码进行登录 |
| 注册页 | 新用户注册账号 |
| 找回密码页 | 通过邮箱验证码重置密码 |
| 主聊天页 | 登录成功后的主界面（包含好友列表、聊天、好友资料、申请管理） |

**页面跳转关系（必须保留）**：

- 登录页 ⇄ 注册页（双向跳转）
- 登录页 ⇄ 找回密码页（双向跳转）
- 登录页 → 主聊天页（登录成功后）
- 主聊天页 → 登录页（被踢下线 / 网络断开 / 主动登出）

## 2. 登录页 功能清单

| # | 功能 | 说明 |
|---|---|---|
| L1 | 输入邮箱 | 必填 |
| L2 | 输入密码 | 必填，长度 6–15 |
| L3 | 提交登录 | 校验通过后向 GateServer 发起登录请求；成功后进入主聊天页，失败后展示错误提示 |
| L4 | 跳转到注册页 | 入口必须存在 |
| L5 | 跳转到找回密码页 | 入口必须存在 |
| L6 | 错误提示展示 | 至少能显示"邮箱格式错误 / 密码格式错误 / 密码不正确 / 网络异常"等服务端返回的错误 |

## 3. 注册页 功能清单

| # | 功能 | 说明 |
|---|---|---|
| R1 | 输入用户名 | 必填 |
| R2 | 输入邮箱 | 必填 |
| R3 | 输入密码 | 必填，长度 6–15 |
| R4 | 输入确认密码 | 必填，必须与密码一致 |
| R5 | 输入邮箱验证码 | 必填，4 位 |
| R6 | 获取验证码 | 点击后向服务端请求发送验证码到邮箱；按钮在请求成功后进入冷却期（防止刷邮件） |
| R7 | 显示/隐藏密码 | 密码框和确认密码框各自可切换可见状态 |
| R8 | 提交注册 | 校验通过后向 GateServer 注册；成功后展示"注册成功"提示并自动返回登录页（约 5 秒倒计时） |
| R9 | 取消 / 返回登录页 | 入口必须存在 |
| R10 | 字段级错误提示 | 用户名 / 邮箱 / 密码 / 确认密码 / 验证码各自有非空与格式校验失败的提示 |

## 5. 主聊天页 功能清单

主聊天页是登录后的核心界面，按"功能区"组织如下。新 UI 可任意重排它们的视觉位置。

### 5.1 当前用户信息展示
| # | 功能 |
|---|---|
| M1 | 展示当前登录用户的头像 |
| M2 | （可选）展示当前用户的昵称 |

### 5.2 模式切换：会话 / 联系人 / 申请
主界面有 3 种"列表模式"，必须能在它们之间切换：

| # | 功能 |
|---|---|
| M3 | 切换到「会话列表」模式 |
| M4 | 切换到「联系人列表」模式 |
| M5 | 切换到「好友申请列表」模式 |
| M6 | 「好友申请」入口需要支持"未读红点"提示（有新申请未处理时显示） |

### 5.3 搜索
| # | 功能 |
|---|---|
| M7 | 输入搜索内容（用户 UID） |
| M8 | 一键清空搜索内容 |
| M9 | 触发搜索：根据 UID 向服务端查询用户，并展示搜索结果 |
| M10 | 搜索结果项：点击后展示"用户存在 → 进入资料页 / 发起加好友"或"未找到"两种结果 |

### 5.4 会话列表
| # | 功能 |
|---|---|
| M11 | 展示当前用户的所有会话条目（头像、对方昵称、最后一条消息预览、未读数） |
| M12 | 点击会话条目：打开右侧「聊天面板」并定位到该会话 |
| M13 | 滚动到底部时自动加载更多会话（分页） |

### 5.5 联系人列表
| # | 功能 |
|---|---|
| M14 | 展示当前用户的所有好友（头像、昵称） |
| M15 | 点击联系人条目：打开右侧「好友资料页」 |
| M16 | 列表中包含一个固定的「新的朋友」入口，点击后切换到「好友申请列表」 |

### 5.6 好友申请列表
| # | 功能 |
|---|---|
| M17 | 展示所有未处理 / 已处理的好友申请（申请人头像、昵称、申请理由、状态） |
| M18 | 对未处理的申请：可点击「同意」入口，弹出"同意好友"对话框 |
| M19 | 对已处理的申请：展示"已添加"状态，不再可点击 |

## 6. 聊天面板 功能清单

主界面右侧"聊天模式"下展示的面板。

| # | 功能 |
|---|---|
| C1 | 顶部展示对方昵称 |
| C2 | 消息历史区：按时间顺序展示双方消息（区分发送方/接收方位置；文本气泡 + 图片气泡 + 文件气泡） |
| C3 | 历史消息懒加载：进入会话时先展示本地缓存的最近一页（约 30 条），再向服务端拉取最新页合并；用户向上滚动到顶部时拉取更早一页 |
| C4 | 实时接收：新消息到达时自动追加到底部 |
| C5 | 输入区：支持文本输入 |
| C6 | 输入区：支持插入图片（从本地选择） |
| C7 | 输入区：支持插入文件（从本地选择） |
| C8 | 发送：点击发送按钮或按 Enter 提交输入区内容；可一次性混合发送多段文本+图片+文件 |
| C9 | 图片消息：在气泡中直接预览缩略图 |
| C10 | 文件消息：展示文件名、大小、类型；对未下载的文件提供下载入口（点击后向服务端申请下载凭证并保存到本地） |

> 注：表情面板按钮、AI 智能回复按钮目前仅有 UI 入口或后端处于接线中，**不属于本次必保留功能**。如新 UI 想预留入口，建议参考 [CLAUDE.md §13](../CLAUDE.md) 的 AgentServer 章节。

## 9. 全局通知 / 状态变化

新 UI 必须能对以下后端事件作出 UI 反馈（呈现方式自由：弹窗、Toast、状态栏皆可）：

| # | 事件 | UI 反馈要求 |
|---|---|---|
| G1 | 收到新好友申请 | 「好友申请」入口出现红点 + 申请列表实时新增条目 |
| G2 | 自己的好友申请被对方同意 | 联系人列表 + 会话列表实时新增该好友 |
| G3 | 收到新消息（且不在当前会话） | 对应会话条目的未读数 +1，最后一条消息预览更新 |
| G4 | 被踢下线（同账号在其他设备登录） | 提示"账号在其他设备登录"，自动返回登录页 |
| G5 | 网络断开（心跳超时 / TCP 异常） | 提示"连接已断开"，自动返回登录页 |
| G6 | 心跳保活 | UI 无需展示，但需在主界面期间持续运行（30 秒一次） |

## 10. 数据来源说明（给 UI 设计师参考）

UI 不需要关心数据怎么来，但设计列表/卡片时需要知道有哪些字段可展示：

- **当前用户**：uid、用户名、昵称、头像 URL、性别、个性签名
- **好友/联系人**：uid、用户名、昵称、头像 URL、备注名、标签、个性签名
- **会话条目**：对方信息（同上）+ 最后一条消息的文本预览 + 未读数 + 最后更新时间
- **消息**：发送者 uid、消息类型（文本 / 图片 / 文件 / 音频）、内容（文本字符串 或 文件 ID + 文件名 + 大小）、发送时间、状态（已发送 / 已读）
- **好友申请**：申请人信息 + 申请理由 + 状态（未处理 / 已同意）

## 11. 不在本次必保留范围的项

下列功能在当前代码中存在 UI 占位但**没有可用的后端实现**，新 UI 可自行决定是否预留入口：

- 表情面板（聊天输入区的笑脸按钮）
- 语音通话 / 视频通话（好友资料页）
- AI 智能回复建议面板（聊天输入区的 AI 按钮，正在 M2 接线）
- 群聊相关一切功能
- 用户个人资料编辑（修改头像 / 昵称 / 签名）
- 消息撤回 / 删除 / 转发
- 消息已读回执的视觉展示

如需在新 UI 中提前预留这些入口，请单独提出，不要影响本表内的核心功能落地。
